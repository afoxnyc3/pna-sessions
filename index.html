Context Architecture

How Clarke currently manages memory across sessions — and where the gaps are relative to the field's best practice (2025–2026). Three targeted additions close the critical gaps without adding new infrastructure. Everything stays in SQLite.

Section 01 — Current State

Today: 5-Layer Memory Injection

Every incoming message triggers buildMemoryContext() — a 5-pass retrieval that prepends context before Claude responds. Memory writes happen asynchronously after each turn.

Critical Gap — Compaction Void
When the context window resets (3 compactions logged), all surfaced memory state is lost. There is no dump, no snapshot, no rebuild path. The next turn starts blind.
High Gap — Flat Structure
All 23 memories live in a single flat table. No hierarchy, no domain grouping, no usage tracking. Time-decay only — memories never earn or lose their place based on usefulness.
Section 02 — Proposed Improvements

Three Targeted Additions

All three stay within the existing SQLite stack. No new infrastructure. No external memory service. Surgical additions that close the field's most validated gaps.

01
Pre-compaction Checkpoint
Closes: CRITICAL
Trigger
Context window hits 70% of max tokens. Checked each turn via token_usage table. Also fires on explicit checkpoint command.
Action
Dump surfaced memory IDs + session decisions to vault (.md note) and memories table tagged sector='checkpoint'. Survives window reset.
Outcome
"Nothing is lost when the window resets." Next session opens with the checkpoint injected as highest-priority context.
02
Usage-Based Salience Scoring
Closes: HIGH
Track
How often each memory is surfaced (retrieved) vs. used (referenced in response). New columns: surface_count, use_count.
Formula
salience = f(importance, recency, use_count, correction_count). Memories cited in responses score up. Memories repeatedly surfaced but ignored score down.
Outcome
"Memories earn their place." Dead memories decay faster. High-signal memories resist decay regardless of age.
03
Lightweight Hierarchical Consolidation
Closes: MEDIUM
Structure
Domain › Category › Memory › Episode
Add domain + category tags to existing memories table. No schema migration needed.
Mechanism
Domain-level consolidation runs alongside flat consolidation. When 5+ memories share a domain tag, synthesize a domain summary. Prevents individual memory noise from polluting category-level signals.
Outcome
"Prevents consolidation rot." Matches H-MEM's index-based routing pattern. Retrieval precision improves as memory count grows.

Section 04 — PRD: New Skill Needed

context-health Skill

The three proposed additions require a lightweight monitoring skill that wraps the checkpoint + salience logic. One skill, three jobs.

Business Problem
  • Context window compactions silently destroy session state — no recovery path exists
  • Memory salience is purely time-based; high-value memories decay alongside low-value ones
  • No automated trigger to save session state before handing off to a principal or ending a long session
  • convolife is manual — no automatic pressure monitoring
Technical Solution
  • New skill: context-health
  • Reads token_usage table to compute % of context window used
  • At 70% threshold: fires pre-compaction checkpoint to vault + SQLite
  • After each turn: increments surface_count / use_count for referenced memories
  • Nightly: re-ranks salience using usage data; updates memories.salience
  • Trigger: context health, how is my context?, auto on threshold
Constraints
  • No new dependencies — stays within SQLite + existing Gemini extraction path
  • Must not block response latency (async where possible)
  • Checkpoint must be idempotent — safe to run multiple times per session
  • Salience re-ranking must not alter the 0.5 importance floor (separate field)
Done Criteria
  • Checkpoint saved to vault within 2 turns of hitting 70% threshold
  • New session opens with checkpoint injected as first memory
  • surface_count and use_count fields populate correctly
  • After 7 days: memory retrieval precision visibly improves (subjective review)
  • Skill documented and passing eval harness
