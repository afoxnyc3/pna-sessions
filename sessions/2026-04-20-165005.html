Research: Extracting SKILL.md Files from Video/Audio Transcripts

Executive Summary

A robust pipeline chains three stages: (1) audio acquisition and transcription via yt-dlp plus Whisper or Deepgram, (2) schema-first structured extraction of procedural atoms from the transcript using map-reduce over chunks with hedge classification to filter uncertain claims, and (3) a modality shift pass that converts first-person narrated events into imperative general procedures formatted as SKILL.md with YAML frontmatter. Prior art exists in fragmented form (ytpipe, TubeScribe, youtube-knowledge-extractor skill, cefriel's procedural-KG pipeline, How2Everything), but no open tool combines all stages end-to-end with a validation harness (skillcheck, skrills-validate, claudelint cover the linting layer).

Key Findings

  1. SKILL.md is a progressive-disclosure format with a strict YAML contract

A SKILL.md folder requires exactly one SKILL.md file (case-sensitive) with YAML frontmatter containing name (kebab-case, must match parent directory) and description. Optional folders: scripts/, references/, assets/. Only the name and description (~200 bytes) are loaded at agent startup; the body loads on match.

Source: YAML Frontmatter - Anthropic Skills and Skill Structure reference -- official and mirrored Anthropic docs.

  1. yt-dlp + Whisper/Deepgram is the dominant transcription stack

Multiple open pipelines (ytpipe, youwhisper-cli, TubeScribe, yt-text) converge on yt-dlp for audio extraction followed by faster-whisper, whisper.cpp, or a cloud engine (Deepgram Nova, OpenAI, Gemini). Deepgram Nova wins on latency and streaming; Whisper large-v3 wins on cost at rest and language breadth. Deepgram's own benchmark claims accuracy and speed advantages but is a vendor document.

Source: TubeScribe repo, ytpipe, Deepgram vs Whisper benchmark (vendor), aimultiple independent benchmark.

  1. Map-reduce is the canonical pattern for long transcripts

Podcasts and conference talks regularly exceed single-context windows. The map step chunks transcript into overlapping windows (typically 1.5k-4k tokens), emits structured candidate steps per chunk, and the reduce step merges, deduplicates, and resolves conflicts into a unified procedure graph. Stuffing fails above ~30 min of audio; refine preserves coherence but costs more.

Source: LangChain summarize_map_reduce docs, Scaling Document Summarization (Soni).

  1. Hedge detection is a well-studied primitive for filtering uncertain claims

Speakers hedge constantly ("I think you could maybe try..."). The CoNLL-2010 shared task and downstream work (HedgeHunter, HEDGEhog) provide token-level classifiers that label spans as epistemic, doxatic, investigation, or condition hedges with ~0.89 F1. Using a hedge classifier as a gate lets the extractor promote only asserted steps to the SKILL.md body and demote hedged content to a "Notes / caveats" section.

Source: "You should probably read this": Hedge Detection in Text, HedgeHunter, HEDGEhog model.

  1. Schema-first extraction with Pydantic or JSON Schema beats freeform generation

Rather than asking the LLM for markdown directly, define a Pydantic model (ProcedureStep with fields: action_verb, object, preconditions, rationale, hedge_level, timestamp, source_quote). The model emits validated JSON; a deterministic renderer converts JSON to SKILL.md. This separation is what Sourcery, LangExtract, and LlamaIndex's PydanticProgramExtractor all implement.

Source: Sourcery: schema-first extraction framework, LlamaIndex PydanticExtractor, Structured Outputs guide.

  1. The narrated-event to general-procedure shift is an abstraction operation

Narrators say "so then I ran npm install and it broke because I had node 18." A SKILL.md requires "1. Verify Node >= 20 before running npm install." The shift has four moves: (a) detach from past tense and first person (strip "I did"), (b) lift particulars to preconditions ("node 18" becomes "older Node versions"), (c) recast failures as guardrails, (d) convert temporal sequence into dependency order. Work on goal-oriented script construction and WikiHow hierarchies formalizes this as script induction from narrative.

Source: Show Me More Details (ACL 2022), Goal-Oriented Script Construction, How2Everything (2026).

  1. Procedural KG extraction gives a sound intermediate representation

Cefriel's prompt-based pipeline extracts steps, actions, objects, equipment, and temporal relations into a typed graph, then serializes. This intermediate KG layer is exactly what you want between the raw transcript and the final SKILL.md: it makes the reduce step a graph merge (not a text diff) and makes validation inspectable.

Source: cefriel/procedural-kg-llm, Human Evaluation paper.

  1. Validation harnesses already exist and enforce cross-agent portability

skillcheck, skill-tools-plugin, skrills-validate, and claudelint all lint SKILL.md: frontmatter schema, name/folder match, description discoverability scoring, token budget tiers, file-reference integrity, and cross-agent compatibility (Claude Code is permissive, Codex CLI and Copilot CLI are strict). A reddit-reported failure mode: skills that load in Claude Code fail silently in VS Code when name does not match the parent directory.

Source: skillcheck, skill-tools-plugin, skrills-validate crate, claudelint.

  1. Direct prior art: youtube-knowledge-extractor skill

The sdrabent/youtube-knowledge-extractor skill on ClawHub/OpenClaw already does multimodal YouTube analysis via transcript plus frame extraction, producing step-by-step guides. It declares yt-dlp, ffmpeg, python3, and curl as required binaries. A simpler cousin abe238/youtube-summarizer only summarizes. Neither performs an explicit hedge-classification pass or emits a nested SKILL.md, so there is an open gap.

Source: youtube-knowledge-extractor on ClawHub, youtube-summarizer SKILL.md.

Deep Dive

Stage 1: Acquisition and transcription. yt-dlp handles YouTube, podcast RSS enclosures, Vimeo, and most other sources via --extract-audio --audio-format wav. If the source already has captions (YouTube auto-captions, conference .vtt), pull them first and skip ASR [Source 4, yt-text]. Otherwise choose a transcriber by workload: local faster-whisper with large-v3 for privacy or scale at rest, Deepgram Nova-3 for speaker diarization and low-latency streaming [Source 10, 11]. Store transcripts as JSON with word-level timestamps so later stages can cite source quotes into the SKILL.md "provenance" section.

Stage 2: Schema-first extraction over map-reduce. Define a Pydantic schema with fields for the atomic procedure step (action, object, tool, precondition, expected outcome, failure mode, hedge level, source timestamp). Chunk the transcript with 10-20% overlap so step boundaries are not severed; LangChain's MapReduceDocumentsChain or a hand-rolled asyncio fanout both work [Source 7]. Map step prompt: "Extract ProcedureStep instances from this chunk; return [] if the chunk is pure narration with no action." Reduce step performs entity alignment, dedupe, and dependency ordering, ideally against a procedural KG intermediate [Source 8, cefriel]. Run hedge classification (HEDGEhog or a GPT-class zero-shot classifier trained on CoNLL-2010 labels) on each extracted step's supporting span [Source 5, 6]. Steps with epistemic/doxatic hedge level above threshold get routed to a ## Caveats section rather than the main imperative body.

Stage 3: Modality shift. This is the linguistically hardest move and the one no existing open tool does well. The input is past-tense first-person narration of a particular event; the output must be present-tense imperative general procedure. Concretely: "I ran npm install and got ERESOLVE" becomes "Run npm install. If ERESOLVE appears, pin peer deps with --legacy-peer-deps or upgrade the conflicting package." This requires four operations grounded in the script-induction literature [Source, ACL 2022 wikihow_hierarchy; How2Everything]: tense/person normalization, variable lifting (proper nouns to role names), failure-to-guardrail conversion, and dependency reordering. Implement it as a dedicated LLM pass with a few-shot prompt containing paired narration/procedure examples, then validate the output contains no first-person pronouns and every verb is imperative or infinitive.

Stage 4: SKILL.md rendering. Deterministic Jinja template consumes the validated JSON/KG and emits:


---
name: <kebab-case derived from video title>
description: <when to use, first-person neutral, <=1024 chars>
version: 0.1.0
source:
  url: <video URL>
  transcribed_by: whisper-large-v3 | deepgram-nova-3
  extracted_at: <ISO timestamp>
---

# <Title>

## When to Use
...

## Prerequisites
...

## Procedure
1. ...
2. ...

## Caveats
<hedged content with source timestamps>

## Provenance
<quoted transcript spans with HH:MM:SS links>

The name field must match the parent directory or VS Code will silently ignore it [Source 9, skillcheck reddit]. Description must be discoverable: include the triggering verbs the agent should match on ("This skill should be used when..." pattern from anthropics/claude-plugins-official skill-development skill).

Stage 5: Validation harness. Run skillcheck or skrills-validate as a post-hook: frontmatter schema, name/folder match, description length, file references exist, token budget within tier. Add project-specific checks: (a) no first-person pronouns in the procedure section (regex), (b) every step has a cited timestamp in Provenance, (c) hedge-classifier reports < X% hedge density on main body. Running the generated skill against a held-out test prompt ("do the thing this skill claims to teach") and scoring the result with an LLM judge gives an end-to-end quality signal analogous to How2Judge in the How2Everything framework [Source, How2Everything].

Prior-art gap. youtube-knowledge-extractor and youtube-summarizer produce guides or summaries but not self-contained SKILL.md files with the schema-first, hedge-gated, modality-shifted pipeline described here. Procedural-kg-llm does the extraction graph but not the SKILL.md output. skillcheck validates but does not generate. The opportunity is a glue skill that orchestrates all five stages.

Feynman Bridge

Note: feynman CLI was unavailable (not installed in this environment); the following is a plain-language synthesis from the cited sources rather than a CLI-grounded explanation.

Think of a talk transcript as a travel diary written in first person: "I took the 8am train, got lost in Rome, finally found the museum at 3pm." A SKILL.md is the opposite: it is a guidebook for anyone who wants to visit the museum. The pipeline is the translator between the two. First you get the diary text (yt-dlp pulls audio, Whisper or Deepgram turns audio into words). Then you read the diary carefully in small chunks because it is too long to read in one breath (map-reduce), picking out every action the author actually performed. Some of what the author says is firm ("I bought a ticket") and some is guesswork ("I think maybe the 2pm one is cheaper"); a hedge classifier sorts those into two piles so the guidebook only tells readers to do what was actually confirmed to work.

The hardest translation step is turning "I got lost, then found the museum" into "Exit the station on the south side and follow Via del Corso." That requires stripping the I, dropping the specific time, and turning a remembered mistake into a general warning. This is the same trick computational linguists use to mine WikiHow: take a narrative and induce the underlying script. The final guidebook gets a structured cover page (YAML frontmatter) so the agent librarian knows when to hand this particular book to a reader, and a linter checks the cover matches the spine (folder name equals the name field) before the book ships.

Actionable Insights

  1. Build the pipeline in five separable stages -- acquire (yt-dlp), transcribe (Whisper/Deepgram), extract (schema-first map-reduce with Pydantic), shift modality (dedicated LLM pass), render + validate (Jinja + skillcheck). Keep each stage behind a JSON boundary so you can swap tools.
  1. Use captions before ASR when they exist -- YouTube auto-captions, conference VTT files, and podcast show-notes-with-transcripts skip a costly step. Fall back to Whisper large-v3 locally or Deepgram Nova-3 for streaming/diarization.
  1. Adopt a Pydantic schema for ProcedureStep early -- fields: action_verb, object, tool, precondition, expected_outcome, failure_mode, hedge_level (enum), source_timestamp, source_quote. Every downstream stage reads and writes this type.
  1. Gate promotion to the main body with a hedge classifier -- route epistemic/doxatic hedged spans to a ## Caveats section with preserved source quotes. Use HEDGEhog or a zero-shot GPT classifier calibrated against CoNLL-2010 labels.
  1. Write an explicit modality-shift prompt -- few-shot with 5-8 narration/procedure pairs, then post-validate that the output contains zero first-person pronouns and every step starts with an imperative verb. This is the move most prior tools skip.
  1. Wire skillcheck or skrills-validate into CI -- enforce name-matches-directory, description discoverability, token budget, and file-reference integrity. Add a project rule that every step in ## Procedure must have a timestamp in ## Provenance.
  1. Evaluate end-to-end with an LLM-judge loop -- following How2Judge, give the generated skill to a fresh agent and ask it to perform the task; score success. Low scores feed back into better extraction prompts.
