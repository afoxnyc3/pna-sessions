02 / agents

The Research Cluster

Three agents, three frameworks, one shared SQLite communication bus. Each runs as a persistent launchd service on the host Mac.

03 / tool layer

Research Capabilities

Three tools shared across both workers. Subprocess-based tools (feynman, alpha) require the fnm node bin prepended to PATH — handled by _tool_env() and each launchd plist.

EXA · WEB SEARCH
Exa API
exa-py >= 2.12.0 · search_and_contents()
General web research. Returns title, URL, published_date, snippet (800 chars), and relevance score for up to 8 results. Uses "auto" search type. Note: use_autoprompt removed in exa-py 2.12.0.
research_a: web_search(query) research_b: web_research(query, num_results=8) Env: EXA_API_KEY
research_a research_b
FEYNMAN · FIRST PRINCIPLES
feynman CLI
feynman research '<query>' --model anthropic/claude-opus-4-7
First-principles explanation of a topic. Grounds the Feynman Bridge section of the research schema. Required for every mission (enforced in research_b system prompt). 120s subprocess timeout — known failure point under load.
feynman research 'query' \ --model anthropic/claude-opus-4-7 # Timeout: 120s # PATH: must include fnm node bin
research_a research_b
ALPHA · ACADEMIC PAPERS
alpha CLI
alpha --json search '<query>' · alphaXiv-backed
Academic paper search via alphaXiv. Returns arXiv papers with title, authors, abstract (600 chars), arxiv_id, URL, year. 60s timeout. Requires alpha login on host machine.
research_a: alpha_search(query) research_b: academic_search(query, num_results=5) # Returns: JSON array of papers # Requires: alpha login
research_a research_b
research_b exclusive
compare_sources
compare_sources(topic, sources[])
Compares multiple retrieved sources on a topic. Takes a topic string and an array of {title, url, snippet} objects. Used to resolve conflicting signals before synthesis.
research_b exclusive
check_novelty
check_novelty(idea)
Checks whether a research idea is genuinely novel against recent literature. Used for idea/approach queries. Part of research_b's expanded 5-tool surface.
04 / infrastructure

Shared Data Layer

SQLite is the only communication layer between all agents. Optimistic locking prevents double-claiming. No message broker, no inter-agent network calls.

05 / output schema

Standard Research Schema

Both agents produce this identical schema. Frontmatter: query · agent · completed_at · confidence. Seven required sections, in order — no section may be skipped.

01
Executive Summary
High-level answer to the research query. Confidence level stated upfront. 3-4 sentences max.
02
Key Findings
Minimum 3 findings, each with a source citation traceable to the Source Ledger.
03
Deep Dive
Detailed technical analysis. Sub-sections allowed. Architecture decisions, trade-offs, implementation specifics. Source-backed.
04
Source Ledger
Table of all sources: title · URL · relevance score · type (paper / blog / docs / official).
05
Feynman Bridge
Plain-language analogy explaining the core concept from first principles. Grounded in feynman CLI output. Required every mission.
06
Actionable Insights
Minimum 3 concrete recommendations. Immediately implementable. No hedging. Ordered by priority.
07
Confidence Assessment
Overall: high / medium / low. Coverage gaps and conflicting signals explicitly stated. Tool failures documented.
